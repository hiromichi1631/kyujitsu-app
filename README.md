# 休日管理ツール

週4日出勤（週休3日）契約向けの個人用休日管理Webアプリです。スマホ利用がメインのレスポンシブデザインで、PWA対応（ホーム画面に追加してアプリのように使えます）。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | アプリ本体（HTML/CSS/JSすべて入り。ビルド不要） |
| `manifest.json` | PWAマニフェスト（アプリ名・アイコン定義） |
| `sw.js` | Service Worker（オフライン対応キャッシュ） |
| `icon-192.png` / `icon-512.png` / `apple-touch-icon.png` | アプリアイコン |

## 主な機能

- **ダッシュボード**: 有給残日数（8月始まりの年度基準）／今月の日曜休み（目標2回）／今週の出勤（目標4日、超過で赤警告）／今月の出勤日数
- **月間カレンダー**: 前後月の日付も表示・編集可、祝日マーク（2026〜2031年内蔵）、今日をハイライト
- **タップ操作**: 休む日だけタップ（休み → 有給 → 出勤に戻る）。火曜は自動で定休（祝日の火曜は営業）、お盆・正月の設定期間は自動で休業
- **月間・年度サマリー**: 出勤/休み/有給/日曜休みの集計、日曜月2回の達成状況グリッド
- **設定**: 有給付与日数、週の出勤目標、お盆・正月休みの期間
- **自動保存**: すべての変更はlocalStorageに自動保存（保存ボタン不要）
- **エクスポート/インポート**: JSON形式でバックアップ・機種変更に対応

## 使い方（ローカルで試す）

`index.html` をダブルクリックで開くだけで使えます（PWA機能を除く全機能が動作）。

## 無料でホスティングする方法（GitHub Pages）

PWA（ホーム画面追加・オフライン動作）はhttps配信が必要なため、ホスティングを推奨します。

1. [GitHub](https://github.com) にアカウントを作成（無料）
2. 「New repository」で新しいリポジトリを作成（例: `kyujitsu-app`、Public を選択）
3. 「uploading an existing file」リンクから、このフォルダ内の全ファイルをドラッグ&ドロップしてアップロード → 「Commit changes」
4. リポジトリの **Settings → Pages** を開き、「Source」を **Deploy from a branch**、Branch を **main / (root)** にして Save
5. 数分後に `https://<ユーザー名>.github.io/kyujitsu-app/` でアプリが公開されます

### スマホのホーム画面に追加

- **iPhone (Safari)**: 公開URLを開く → 共有ボタン → 「ホーム画面に追加」
- **Android (Chrome)**: 公開URLを開く → メニュー(⋮) → 「ホーム画面に追加」または「アプリをインストール」

### その他の無料ホスティング

- **Netlify Drop** (https://app.netlify.com/drop): フォルダをドラッグ&ドロップするだけで公開。GitHubアカウント不要で最も手軽
- **Cloudflare Pages** / **Vercel**: GitHubリポジトリと連携して自動デプロイ

## データについて

- データは端末のブラウザ内（localStorage）にのみ保存されます。サーバーには送信されません
- ブラウザのデータ消去や機種変更で消えるため、定期的に「エクスポート」でバックアップを取ることをおすすめします
- 祝日データは2031年まで内蔵。以降は `index.html` 内の `HOLIDAYS` に追記してください
